1. Features

  • The product is an Internet Protocol (IP) multi-protocol tester at layer 4; so we are able to support only IP based protocols.
    MTS has been designed specially for IMS architecture and we of course manage the main IMS protocols : SIP, RTP, Diameter, HTTP.
    Moreover it also supports a lot of other application protocols like Radius, RTSP, SMTP, IMAP, POP, SMPP, UCP … developed on customer demand.
    And we finally provide the support of the 4 main transport protocols: UDP, TCP, TLS and SCTP.

  • With the MTS product, you are able to perform :
    * protocol, functional, non-regression tests (sequential mode)
    * Load endurance and stress tests (load mode)
    * Simulation of the client, server or both sides

  • The user interacts with MTS using 2 different interfaces :
    * A graphical one (for test writing and tuning)
    * A command-line (for load or automatic testing)

  • The tests are described in XML files with a test and many scenarios input files.
    The product is easy to use because of logging management and rich statistics presentation.

  • MTS is a software application entirely written in java and so you can run it on a lot of commercial platforms (support on Windows system, Linux madriva distribution); there is no need to buy expensive hardware.

2. Requirements

    The MTS needs to install the following third party products :

  • Linux Mandriva or Windows XP operating system (no specific hardware is required)

  • Java Runtime Environment (JRE) version 1.5 or 1.6 that may be downloaded from http://java.sun.com/downloads/.
    To have better performance during load testing mainly, we recommand to use the JRE 1.6 version.

  • lksctp-tools version 1.0.4 SCTP library, if you want to use the SCTP protocol (only supported on Linux kernel version 2.6).

  • jpcap version 0.7 to use the capture possibility on MTS. jpcap can be found at http://netresearch.ics.uci.edu/kfujii/jpcap/doc/download.html.
    Explaination on jpcap needs and installation information for each platform can be found at http://netresearch.ics.uci.edu/kfujii/jpcap/doc/install.html.

3. Installation :

    Here is the procedure to install the product on your computer :

  • 1) Install first the JRE version 1.5 or 1.6.

  • 2) Execute the installer archive : java –jar imsloader_V[x.y.z]_setup_[customer].jar
    where [x.y.z] is the product version and [customer] is your customer reference.

4. Quick startup :

    Execute the following shell commands to start the GUI user interface:

  • 1) to go into the installation directory : cd [installDir]/bin.

  • 2) run in a command windows : startGui.bat on Windows platform.

  • 3) run in a terminal : ./startGui.sh on Linux environment.
